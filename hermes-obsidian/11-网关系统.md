# 网关系统

> 21 平台消息路由与 Session 管理

## 概述

网关是 Hermes 的多平台消息集成层，支持 Telegram、Discord、Slack、WhatsApp、微信等 21 个平台。GatewayRunner 编排所有平台适配器，管理 Session 缓存和消息路由。

## 核心架构

```
GatewayRunner (run.py:620)
  ├── Platform Adapters (21 platforms)
  ├── Channel Directory (可达频道映射)
  ├── Delivery Router (消息路由)
  ├── Session Mirror (跨平台投递)
  └── Hook System (事件钩子)
```

### 平台枚举 (`gateway/config.py:48-70`)

LOCAL, TELEGRAM, DISCORD, WHATSAPP, SLACK, SIGNAL, MATTERMOST, MATRIX, HOMEASSISTANT, EMAIL, SMS, DINGTALK, API_SERVER, WEBHOOK, FEISHU, WECOM, WECOM_CALLBACK, WEIXIN, BLUEBUBBLES, QQBOT, YUANBAO

## 实现原理

### BasePlatformAdapter ABC (`gateway/platforms/base.py:1081`)

每个平台必须实现:
```python
connect() -> bool           # 建立连接
disconnect()                # 断开连接
send(chat_id, content)      # 发送文本
get_chat_info(chat_id)      # 查询聊天元数据
```

可选覆盖: `edit_message()`, `delete_message()`, `send_typing()`, `send_image()`, `send_voice()`, `play_tts()`, `send_video()`, `send_document()`

**Session 状态追踪**:
- `_active_sessions` — interrupt Events 字典
- `_pending_messages` — 消息队列
- `_session_tasks` — 任务句柄

**消息到达时**: 如果 session 活跃 → 排队（照片突发）或信号中断。斜杠命令 (`/stop`, `/new`, `/reset`) 绕过活跃 session 保护。

### MessageEvent (line 826)

```python
@dataclass
class MessageEvent:
    source: SessionSource
    content: str
    message_type: MessageType  # TEXT/PHOTO/VOICE/VIDEO/DOCUMENT/STICKER/ANIMATION/COMMAND
    message_id: str
    media: List[MediaAttachment]
    reply_to: Optional[str]
    metadata: Dict
```

### SessionSource (`gateway/session.py:70`)

```python
@dataclass
class SessionSource:
    platform: Platform
    chat_id: str
    chat_name: str
    chat_type: ChatType  # dm/group/channel/thread
    user_id: str
    user_name: str
    thread_id: Optional[str]
    guild_id: Optional[str]
    message_id: str
```

### 消息路由

#### Channel Directory (`gateway/channel_directory.py`)
- 构建每平台可达频道的缓存映射
- Discord: 枚举 guild text/forum 频道
- Slack: `users.conversations` API
- 其他: 从 session 历史发现
- 缓存: `~/.hermes/channel_directory.json`, 5 分钟刷新

#### Delivery Router (`gateway/delivery.py`)
路由目标格式:
- `"origin"` — 返回来源
- `"local"` — 保存到文件
- `"telegram"` — 主频道
- `"telegram:123456"` — 特定聊天

超过 4000 字符截断 + 链接到完整输出文件

#### Session Mirror (`gateway/mirror.py`)
跨平台投递时，`delivery-mirror` 记录追加到目标 session 的 JSONL 和 SQLite

### GatewayRunner (`gateway/run.py:620`)

- Agent 缓存: `OrderedDict`, 上限 128, 1 小时空闲 TTL（prompt caching 效率）
- `start()` (line 2106): 加载配置 → 发现插件 → 注册 shell hooks → 发现事件 hooks → 恢复后台进程 → 构建 channel directory → 启动所有适配器
- `_handle_message()` (line 3347): 解析 session → 构建/获取缓存 AIAgent → 分发

**Session 重置策略**: daily(特定小时) / idle(N分钟空闲) / both / none

### Hook System (`gateway/hooks.py`)

从 `~/.hermes/hooks/` 发现事件钩子:
- 每个钩子目录: `HOOK.yaml` (元数据) + `handler.py` (处理器)
- 事件: `gateway:startup`, `session:start/end/reset`, `agent:start/step/end`, `command:*`
- 内置 `boot-md` 钩子: 启动时运行 `~/.hermes/BOOT.md`

### 平台实现

#### Telegram (`gateway/platforms/telegram.py`)
- 使用 `python-telegram-bot`
- MarkdownV2 格式化 + 特殊字符转义
- GFM 管道表 → 代码块（Telegram 无表格语法）
- 内联键盘、回调查询
- `telegram_network.py` 提供 IP 发现的备用传输

#### 微信 (`gateway/platforms/weixin.py`)
- 腾讯 iLink Bot API
- 长轮询 `getupdates`
- AES-128-ECB 加密 CDN 媒体协议
- QR 登录

## 数据流

```
平台消息到达
  ↓
PlatformAdapter.handle_message(event)
  ↓
Session 解析 (chat_id → session)
  ↓
GatewayRunner._handle_message()
  ├── 获取/创建 AIAgent (128 缓存)
  ├── AIAgent.run_conversation()
  └── response
  ↓
Delivery Router → 目标平台
```
