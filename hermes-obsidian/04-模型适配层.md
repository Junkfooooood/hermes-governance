# 模型适配层

> 屏蔽 15+ LLM 提供商差异的统一抽象层

## 概述

模型适配层通过 Transport 抽象和 Adapter 模式，将不同 LLM 提供商（Anthropic、OpenAI、Google、Bedrock 等）的 API 差异统一为标准接口，并通过凭证池实现高可用。

## 核心架构

### Transport 抽象层

```
ProviderTransport (ABC)
    ├── AnthropicTransport   (anthropic_messages)
    ├── CodexTransport       (codex_responses)
    ├── ChatCompletionsTransport (chat_completions)
    └── BedrockTransport     (bedrock_converse)
```

**注册机制**: `register_transport(api_mode, cls)` / `get_transport(api_mode)`
自动发现：导入 `agent/transports/{anthropic,codex,chat_completions,bedrock}.py`

### ProviderTransport ABC (`agent/transports/base.py`)

```python
class ProviderTransport:
    def convert_messages(messages)      # OpenAI 格式 → 提供商原生格式
    def convert_tools(tool_definitions) # 工具定义转换
    def build_kwargs(...)               # 构建完整 API 调用参数
    def normalize_response(raw)         # 原始响应 → NormalizedResponse
```

### NormalizedResponse (`agent/transports/types.py`)

标准化响应形状：
- `content` — 文本内容
- `tool_calls: List[ToolCall]` — 工具调用列表
- `finish_reason` — 结束原因
- `reasoning` — 推理内容
- `usage` — token 用量
- `provider_data: Dict` — 协议特定状态（Anthropic reasoning_details, Codex items 等）

## 实现原理

### 适配器文件

| 适配器 | 文件 | 大小 | 用途 |
|--------|------|------|------|
| Anthropic | `agent/anthropic_adapter.py` | 72KB | Claude 原生 API |
| Codex | `agent/codex_responses_adapter.py` | 45KB | OpenAI Responses API |
| Bedrock | `agent/bedrock_adapter.py` | 48KB | AWS Bedrock |
| Gemini Native | `agent/gemini_native_adapter.py` | 34KB | Google Gemini |
| Gemini CloudCode | `agent/gemini_cloudcode_adapter.py` | 34KB | Gemini via Cloud Code |

### 凭证池 `CredentialPool` (`agent/credential_pool.py`)

**PooledCredential** (line 91) — 每凭证状态追踪：
- provider, auth_type (oauth/api_key)
- priority, access/refresh tokens
- exhaustion 状态, error codes
- cooldown 时间戳, request count

**选择策略** (line 63):
| 策略 | 说明 |
|------|------|
| `fill_first` | 优先使用第一个可用凭证 |
| `round_robin` | 轮询 |
| `random` | 随机 |
| `least_used` | 最少使用 |

**耗尽处理**: 429 错误冷却 1 小时，或使用提供商返回的 reset 时间戳

### Anthropic 特殊处理

- 支持 API Key、OAuth setup-tokens、Claude Code 凭证
- effort level 映射到自适应思考：`xhigh`/`high`/`medium`/`low`
- 每模型输出限制表

### 支持的提供商

Nous, OpenRouter, OpenAI-Codex, Copilot (ACP), Gemini, ZAI (Zhipu), Kimi-Coding, MiniMax, HuggingFace, Arcee, StepFun, AI-Gateway, Kilocode, OpenCode-Zen, OpenCode-Go + 自定义 OpenAI 兼容端点

## 数据流

```
AIAgent._run_agent_loop()
  ↓
get_transport(api_mode)
  ↓
transport.convert_messages(messages)   # 格式转换
transport.convert_tools(tools)         # 工具定义转换
transport.build_kwargs(...)            # 构建参数
  ↓
CredentialPool.get_credential()        # 获取凭证
  ↓
API 调用 (httpx)
  ↓
transport.normalize_response(raw)      # 标准化响应
  ↓
NormalizedResponse → AIAgent 处理
```

## 配置项

```yaml
# config.yaml
model: "claude-sonnet-4-20250514"
providers:
  anthropic:
    api_key: ${ANTHROPIC_API_KEY}
  openrouter:
    api_key: ${OPENROUTER_API_KEY}
credential_pool_strategies:
  openrouter: round_robin
  anthropic: fill_first
fallback_providers:
  - provider: openrouter
    model: anthropic/claude-sonnet-4-20250514
```
