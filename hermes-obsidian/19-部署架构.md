# 部署架构

> Docker 容器化部署与 entrypoint 机制

## 概述

Hermes 支持通过 Docker 容器化部署，包含 gateway 和 dashboard 两个服务。entrypoint 脚本处理 UID/GID 映射、目录初始化和权限管理。

## 核心架构

```
docker-compose.yml
  ├── gateway 服务 (hermes gateway run)
  └── dashboard 服务 (hermes dashboard)

Dockerfile
  ├── debian:13.4 基础镜像
  ├── uv 包管理器
  ├── Node.js + npm
  ├── Playwright Chromium
  └── Python 虚拟环境
```

## 实现原理

### Dockerfile

- **基础镜像**: `debian:13.4`
- **工具**: uv (包管理器), gosu (权限降级), tini (僵尸进程回收)
- **系统依赖**: build-essential, nodejs, npm, python3, ripgrep, ffmpeg, gcc, git, openssh-client, docker-cli
- **非 root 用户**: `hermes` (UID 10000)
- **Playwright Chromium**: 浏览器工具
- **Web Dashboard**: Vite 构建
- **Python 环境**: `uv pip install -e ".[all]"`
- **入口**: `/usr/bin/tini -g -- /opt/hermes/docker/entrypoint.sh`

### Entrypoint (`docker/entrypoint.sh`)

以 root 运行时:
1. 重映射 hermes UID/GID 匹配宿主 (`HERMES_UID`/`HERMES_GID`)
2. 修复卷所有权
3. 通过 `gosu` 降权

创建必要目录: cron, sessions, logs, hooks, memories, skills, skins, plans, workspace, home

初始化: 种子 `.env`, `config.yaml`, `SOUL.md` (从示例) + 同步内置技能

### docker-compose.yml

| 服务 | 命令 | 网络 | 说明 |
|------|------|------|------|
| gateway | `hermes gateway run` | host | 挂载 `~/.hermes` → `/opt/data` |
| dashboard | `hermes dashboard --host 127.0.0.1 --no-open` | — | 仅 localhost |

安全: dashboard 绑定 127.0.0.1，API 服务器默认关闭（需 `API_SERVER_KEY`）。远程访问推荐 SSH 隧道。

## 配置项

```bash
# 环境变量
HERMES_UID=$(id -u)  # 宿主 UID 映射
HERMES_GID=$(id -g)  # 宿主 GID 映射
```

```yaml
# docker-compose.yml
services:
  gateway:
    image: hermes-agent
    network_mode: host
    volumes:
      - ~/.hermes:/opt/data
  dashboard:
    image: hermes-agent
    command: dashboard --host 127.0.0.1 --no-open
    depends_on:
      - gateway
```
