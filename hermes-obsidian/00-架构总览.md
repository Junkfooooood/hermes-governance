# Hermes Agent 架构总览

## 项目简介

Hermes Agent 是 Nous Research 开发的**自进化多平台 AI 代理系统**（v0.11.0），具备分层记忆、多代理治理、21 平台消息网关和 RL 训练能力。

## 三层架构

```
┌─────────────────────────────────────────────────┐
│                   网关层 (Gateway)                │
│   Telegram / Discord / Slack / WeChat / ...      │
├─────────────────────────────────────────────────┤
│               Agent 核心引擎 (run_agent.py)       │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐            │
│  │模型   │ │工具   │ │记忆   │ │治理   │            │
│  │适配层 │ │系统   │ │系统   │ │框架   │            │
│  └──────┘ └──────┘ └──────┘ └──────┘            │
├─────────────────────────────────────────────────┤
│               运行时配置层 (~/.hermes)             │
│  SOUL.md / memory.md / skills / fact_store /     │
│  harness / byterover / neo4j / qdrant            │
└─────────────────────────────────────────────────┘
```

## 数据流概览

```
用户消息 (Telegram/CLI/...)
    ↓
GatewayRunner._handle_message()
    ↓
AIAgent.run_conversation()
    ├── _build_system_prompt()  ← SOUL.md + MEMORY.md + 技能索引
    ├── _memory_manager.prefetch_all()  ← 记忆预取
    ├── while loop:
    │   ├── LLM API call  ← Transport 层
    │   ├── response.tool_calls  ← 模型返回工具调用
    │   ├── _execute_tool_calls()  ← Registry dispatch
    │   └── append results
    └── _memory_manager.sync_turn()  ← 记忆持久化
    ↓
Delivery Router → 目标平台
```

## 关键设计哲学

1. **指针优先**: 黄金层只存指针，不存正文，减少上下文噪声
2. **新陈代谢**: 记忆有升级/降级/归档机制，重要的逐渐固化
3. **降噪音**: 噪声超过 30% 触发修复，超过 50% 强制压缩
4. **反馈闭环**: 从现实世界自动学习，验证假设，更新认知
5. **安全第一**: 7 条不可变宪法法则，无确认不破坏
