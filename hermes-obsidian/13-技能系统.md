# 技能系统

> 过程性记忆：发现、渐进式披露、自动合并与淘汰

## 概述

技能是 Hermes 的"过程性记忆"——捕获"如何做某类任务"的可复用程序。技能有完整的生命周期：发现 → 列表 → 查看 → 加载 → 执行，以及自动合并和淘汰机制。

## 核心架构

```
Skills Tool (tools/skills_tool.py)
  ├── skills_list — 元数据列表（Tier 1）
  └── skill_view — 完整内容（Tier 2/3）

Skill Manager (tools/skill_manager_tool.py)
  └── create/edit/patch/delete/write_file/remove_file

Skills Hub (tools/skills_hub.py)
  ├── OptionalSkillSource — 官方可选技能
  ├── GitHubSource — GitHub 仓库技能
  └── HubLockFile — 来源追踪
```

## 实现原理

### 技能定义

目录结构: `~/.hermes/skills/<category>/<skill-name>/`
```
SKILL.md          # YAML frontmatter + 指令
references/       # 参考文档
templates/        # 模板文件
scripts/          # 脚本
assets/           # 资源
```

SKILL.md frontmatter:
```yaml
name: skill-name
description: 一句话描述
version: 1.0.0
license: MIT
platforms: [macos, linux]
prerequisites: [env_var, tool]
compatibility: {min_version: 0.10.0}
metadata: {category, tags}
```

### 渐进式披露 (Progressive Disclosure)

| 层级 | 内容 | 场景 |
|------|------|------|
| Tier 1 | 名称 + 描述 | `skills_list` — token 高效 |
| Tier 2 | 完整指令 | `skill_view("name")` |
| Tier 3 | 链接文件 | `skill_view("name", "references/api.md")` |

### Skill Manager

允许代理创建/更新/删除技能。技能是代理的**程序性记忆**——基于经验证明捕获"如何做某类任务"。

安全扫描: `skills_guard` 可用但默认关闭

### Skills Hub

- `GitHubAuth`: 共享 GitHub API 认证 (PAT, gh CLI, GitHub App)
- `SkillSource` ABC: 所有技能注册适配器接口
- `OptionalSkillSource`: 官方可选技能
- `GitHubSource`: 从任何 GitHub 仓库获取技能（Contents API）
- `HubLockFile`: 追踪已安装 Hub 技能的来源
- 支持隔离、审计日志、taps、索引缓存

### 技能治理 (`skills/governance.md`)

**4 维评分**（不同于记忆评分）:
| 维度 | 权重 | 说明 |
|------|------|------|
| Usage Frequency | 0.40 | 调用次数/创建天数 |
| Success Rate | 0.30 | 成功次数/总调用 |
| Recency | 0.20 | 1.0 - (距上次使用天数/90) |
| Uniqueness | 0.10 | 1.0 - max_overlap_ratio |

**分数解读**: ≥0.70 健康 / 0.40-0.69 风险 / <0.40 不健康

**重叠检测**: tag_jaccard*0.30 + desc_similarity*0.50 + tool_jaccard*0.20
阈值 ≥0.70 触发合并建议

## 数据流

```
代理需要执行任务
  ↓
skills_list → Tier 1 元数据
  ↓
skill_view("name") → Tier 2 完整指令
  ↓
加载 (env var 检查, 平台过滤, 就绪状态)
  ↓
作为注入指令执行
```
