# 工具系统

> 40+ 工具的注册、发现与执行框架

## 概述

工具系统通过单例 `ToolRegistry` 管理所有工具的生命周期，支持 AST 自动发现、并发执行、插件钩子和 MCP 扩展。

## 核心架构

```
ToolRegistry (单例)
  ├── ToolEntry[] (name, schema, handler, check_fn)
  ├── discover_builtin_tools() — AST 扫描
  └── dispatch(name, args) — 执行调度

model_tools.py (编排层)
  ├── get_tool_definitions() — 解析工具集、过滤、重建 schema
  └── handle_function_call() — 主调度器

toolsets.py (工具集定义)
  └── _HERMES_CORE_TOOLS — CLI 完整工具集列表
```

## 实现原理

### ToolRegistry (`tools/registry.py`)

**ToolEntry** 结构：
```python
@dataclass
class ToolEntry:
    name: str                    # 工具名
    toolset: str                 # 所属工具集
    schema: Dict                 # OpenAI function-call 格式
    handler: Callable            # 处理函数
    check_fn: Callable           # 可用性检查
    requires_env: List[str]      # 依赖环境变量
    is_async: bool               # 是否异步
    emoji: str                   # 显示图标
    max_result_size_chars: int   # 结果最大字符数
```

### 工具发现机制

`discover_builtin_tools()` (line 56)：
1. 扫描 `tools/*.py`
2. AST 解析 `_module_registers_tools()` — 检查是否有顶层 `registry.register()` 调用
3. 导入匹配模块触发注册

这是一种**惰性、导入安全**的方式 — 只有包含注册调用的模块才会被导入。

### 工具注册示例

```python
# tools/web_tools.py (line 2080)
registry.register(
    name="web_search",
    toolset="web",
    schema=WEB_SEARCH_SCHEMA,
    handler=lambda args, **kw: web_search_tool(args.get("query", ""), limit=5),
    check_fn=check_web_api_key,
    requires_env=_web_requires_env(),
    emoji="...",
    max_result_size_chars=100_000,
)
```

### 工具调度

`registry.dispatch(name, args, **kwargs)` (line 292)：
1. 查找 ToolEntry
2. 桥接异步处理器 `_run_async()`
3. 捕获所有异常返回 `{"error": "..."}`

### 编排层 `model_tools.py`

- `get_tool_definitions()` (line 203)：解析工具集 → 过滤 check_fn → 重建动态 schema → 后端兼容性清理
- `handle_function_call()` (line 494)：类型强制 → 插件钩子 → registry.dispatch()

### 并发执行

`_execute_tool_calls()` 分发到：
- `_execute_tool_calls_sequential` — 串行
- `_execute_tool_calls_concurrent` — `ThreadPoolExecutor` max 8 workers

并发安全检测 `_should_parallelize_tool_batch()` (line 312)：
- 工具在 `_PARALLEL_SAFE_TOOLS` 白名单中
- 或路径作用域非重叠

## 工具分类

| 类别 | 工具 | 文件 |
|------|------|------|
| 终端 | terminal (6 后端) | `tools/terminal_tool.py` |
| 文件 | read_file, write_file, patch, search_files | `tools/file_tools.py` |
| 网络 | web_search, browse (4 后端) | `tools/web_tools.py` |
| 浏览器 | browser, browser_cdp, browser_camofox | `tools/browser_*.py` |
| 代理 | delegate_task | `tools/delegate_tool.py` |
| 记忆 | memory (MEMORY.md/USER.md) | `tools/memory_tool.py` |
| 技能 | skills_list, skill_view, skill_manager | `tools/skills_*.py` |
| MCP | 外部 MCP 服务器工具 | `tools/mcp_tool.py` |
| 语音 | tts, transcribe | `tools/tts_tool.py`, `tools/transcription_tools.py` |
| 安全 | approval (危险命令审批) | `tools/approval.py` |
| 中断 | interrupt (全局中断) | `tools/interrupt.py` |

## 工具集 (Toolsets)

`toolsets.py` — 命名工具组：
- `_HERMES_CORE_TOOLS` — CLI 完整工具集
- 独立工具集：`web`, `terminal`, `file`, `browser`, `skills`, `vision`, `image_gen`, `moa`, `cronjob`, `messaging`, `rl`
- MCP 工具注册为 `mcp-<server>` 前缀

## 配置项

```yaml
# config.yaml
toolsets:
  - hermes-cli
  # - web
  # - browser
  # - vision
terminal:
  backend: local
  timeout: 180
web:
  backend: firecrawl
```
