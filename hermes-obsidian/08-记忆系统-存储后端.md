# 记忆系统 — 存储后端

> Redis + Qdrant + Neo4j + AMS 四服务记忆栈

## 概述

完整记忆系统依赖四个 Docker 服务：Redis（元数据/FTS）、Qdrant（向量检索）、Neo4j（关系图谱）、AMS（记忆治理编排）。Byterover 提供本地语义索引和时间线检索。

## 核心架构

```
┌─────────────┐
│  AMS API    │ ← 记忆治理编排 (port 8000)
├─────────────┤
│ Qdrant      │ ← 向量相似度搜索 (port 6333/6334)
├─────────────┤
│ Neo4j       │ ← 关系图谱 (port 7474/7687)
├─────────────┤
│ Redis Stack │ ← 元数据/FTS5/工作记忆 (port 6379)
└─────────────┘
```

## 实现原理

### Redis Stack (`redis/redis-stack:latest`)

- 元数据存储
- FTS5 全文搜索
- 工作记忆

### Qdrant (`qdrant/qdrant:latest`)

3 个集合，均使用 1536 维余弦向量（text-embedding-3-small）:

#### hermes_memories
Payload 索引: memory_id(keyword), type(keyword), layer(keyword), importance(float), stability(float), confidence(float), tags(keyword), source(keyword), timestamp(datetime)

HNSW 配置: m=16, ef_construct=100

#### hermes_facts
Payload 索引: fact_id(keyword), category(keyword), subcategory(keyword), tags(keyword), importance(float)

#### hermes_events
Payload 索引: event_id(keyword), type(keyword), timestamp(datetime), importance(float), tags(keyword), entities(keyword)

**Embedding 生成**: 前缀类型和层上下文 `"[{type}] [{layer}] {content_summary}"`

**重排策略**:
```
top-20 余弦相似度
  → 时间加成 +10%
  → 重要性加成 +20%
  → 层级优先 (core > long_term > short_term)
  → 标签重叠 +15%
  → 返回 top-10
```

### Neo4j (`neo4j:2026.01.3`)

6 种节点类型:
| 节点 | 属性 |
|------|------|
| Entity | id, type, name, ticker, description |
| Event | timestamp, importance, source |
| Thesis | confidence, status, expires, memory_pointer |
| Decision | rationale, outcome |
| Feedback | expected, actual, gap_analysis, impact |
| Indicator | frequency, current_value, trend |

9 种关系类型:
| 关系 | 用途 |
|------|------|
| CAUSED_BY | 事件因果 |
| SUPPORTS | 事件支持论点 |
| CONTRADICTS | 事件反驳论点 |
| PRECEDES | 事件时序 |
| EVOLVED_FROM | 论点演化 |
| BASED_ON | 论点依据 |
| LED_TO | 决策链 |
| ABOUT | 关联实体 |
| GENERATED | 生成反馈 |
| UPDATED | 反馈更新论点 |

**Cypher 查询模板**: 时间线重建、因果链追溯（shortestPath）、论点演化、反馈影响分析、决策追踪、跨实体分析

### Byterover 本地索引

#### 语义索引 (`byterover/semantic_index/`)

- Embedding 模型: text-embedding-3-small, 1536 维
- 3 个集合: hermes_memories, hermes_facts, hermes_events
- 重排权重: recency 0.10, importance 0.20, layer_priority 0.15
- 去重阈值: 0.95 余弦相似度

**检索流**: embed query → search hermes_memories (layer+importance filter) → 不足则 search hermes_facts → 仍不足 search hermes_events → 合并重排 → top-10 + 指针

#### 时间线索引 (`byterover/timeline_index/`)

- 按年/月组织的时序事件记录
- 每月一个 YAML 文件，最多 1000 事件
- 默认搜索范围: 90 天，最大 3650 天
- 730 天后归档

**查询模式**: 时间范围、实体时间线、事件序列（沿 related_events 回溯，交叉引用 Neo4j 因果关系）

#### 事件 Schema (`byterover/events/schema.md`)

10 种事件类型: earnings, macro_data, policy_change, news, market_move, thesis_update, decision, feedback, research_milestone, system_event

事件生命周期: 发生 → timeline_index 记录 → Neo4j 插入 → Qdrant 嵌入 → 时间线查询可用

事件**不可变**；修正创建新事件通过 CONTRADICTS 链接

### AMS Docker 服务 (`redislabs/agent-memory-server:latest`)

- 记忆治理编排
- 端口 8000
- 需要 `OPENAI_API_KEY` 用于 embeddings

## 数据流

```
记忆写入:
  AMS 评分 → 路由决策
    → Qdrant (向量)
    → Neo4j (关系)
    → Redis (元数据)
    → Byterover (本地索引)

记忆检索:
  1. 指针层 (SOUL.md + memory.md)
  2. 关键词 (fact_store + skills)
  3. 语义 (byterover + Qdrant)
  4. 图/时间线 (Neo4j)
```

## 配置项

```bash
# hermes-stack/.env
OPENAI_API_KEY=sk-xxx
NEO4J_PASSWORD=your_password
```

```yaml
# docker-compose.yml (hermes-stack/)
services:
  redis:     # redis/redis-stack:latest
  qdrant:    # qdrant/qdrant:latest
  neo4j:     # neo4j:2026.01.3
  ams:       # redislabs/agent-memory-server:latest
```
