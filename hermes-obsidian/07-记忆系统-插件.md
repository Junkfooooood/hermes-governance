# 记忆系统 — 插件

> 8 个可插拔记忆提供商的实现细节

## 概述

Hermes 支持 8 个记忆插件，同时只能激活一个（与内置记忆并存）。插件通过 `MemoryProvider` ABC 接入，支持语义搜索、知识图谱、向量检索等高级记忆能力。

## 核心架构

### 插件发现 (`plugins/memory/__init__.py`)

`_ProviderCollector` 类捕获 `register_memory_provider` 调用

- `discover_memory_providers()` → `(name, description, is_available)` 元组
- `load_memory_provider(name)` → 通过 bundled 或 user-installed 目录解析
- `_is_memory_provider_dir()` → 文本扫描 `__init__.py` 中的注册模式

### MemoryProvider ABC (`agent/memory_provider.py`)

生命周期方法:
```python
class MemoryProvider:
    name: str                           # 短标识符
    is_available() -> bool              # 检查配置/凭证（无网络调用）
    initialize(session_id, **kwargs)    # 创建资源
    system_prompt_block() -> str        # 系统提示词中的静态文本
    prefetch(query, session_id)         # 每次 API 调用前召回
    queue_prefetch(query, session_id)   # 后台预取下一轮
    sync_turn(user, assistant, session) # 持久化已完成轮次
    get_tool_schemas() -> List[Dict]    # OpenAI function calling 格式
    handle_tool_call(name, args)        # 调度工具调用
    shutdown()                          # 清理退出
```

可选钩子: `on_turn_start()`, `on_session_end()`, `on_pre_compress()`, `on_delegation()`, `on_memory_write()`

## 实现原理

### ByteRover (`plugins/memory/byterover/`)

**类**: `ByteRoverMemoryProvider`

- 使用 `brv` CLI 的持久化知识树，层级化上下文
- **本地优先** + 可选云端同步 (`BRV_API_KEY`)
- 工作目录: `$HERMES_HOME/byterover/`

**3 个工具**:
| 工具 | 功能 |
|------|------|
| `brv_query` | 搜索知识树 |
| `brv_curate` | 存储事实/决策/模式 |
| `brv_status` | CLI 版本/树统计/同步状态 |

**关键行为**:
- `prefetch()` 同步运行（10s 超时）
- `sync_turn()` 后台线程整理对话
- `on_memory_write()` 镜像内置记忆写入
- `on_pre_compress()` 上下文压缩前提取洞察

### Holographic (`plugins/memory/holographic/`)

**类**: `HolographicMemoryProvider`

- 本地 SQLite 事实存储 + FTS5 搜索 + 信任评分 + 实体解析
- **HRR（全息缩减表示）** 组合检索

**2 个工具**:
| 工具 | 动作 |
|------|------|
| `fact_store` | add, search, probe, related, reason, contradict, update, remove, list |
| `fact_feedback` | 评分 helpful/unhelpful 训练信任分 |

**HRR 实现** (`holographic.py`):
- **相位向量编码**: SHA-256 确定性原子生成
- `bind(a, b)` = 循环卷积（相位加法）
- `unbind(memory, key)` = 循环相关（相位减法）
- `bundle(*vectors)` = 复数指数循环均值叠加
- `similarity(a, b)` = 相位余弦相似度
- `snr_estimate()` = sqrt(dim/n_items)，SNR < 2.0 时警告

**SQLite Schema** (`store.py`):
- `facts` — fact_id, content(UNIQUE), category, tags, trust_score(0.5), hrr_vector(BLOB)
- `entities` — entity_id, name, entity_type, aliases
- `fact_entities` — 关系表
- `memory_banks` — bank_name, vector(BLOB), dim, fact_count
- `facts_fts` — FTS5 虚拟表 + 同步触发器

**信任调整**: helpful +0.05, unhelpful -0.10, 钳制 [0.0, 1.0]

**混合检索** (`retrieval.py`):
```
FTS5 候选 (limit*3) → Jaccard 重排 → 信任加权 → 时间衰减(可选)
权重: FTS 0.4, Jaccard 0.3, HRR 0.3
```

5 种检索策略:
| 策略 | 说明 |
|---|---|
| `search()` | 混合 FTS5 + Jaccard + HRR |
| `probe(entity)` | HRR 代数实体查询 |
| `related(entity)` | 结构关联发现 |
| `reason(entities)` | 多实体组合查询（AND 语义） |
| `contradict()` | 自动矛盾检测（高实体重叠 + 低内容相似） |

### Mem0 (`plugins/memory/mem0/`)

**类**: `Mem0MemoryProvider`

- 服务端 LLM 事实提取 + 语义搜索 + 重排 + 自动去重
- 通过 Mem0 Platform API

**3 个工具**:
| 工具 | 功能 |
|------|------|
| `mem0_profile` | 所有存储记忆（无重排） |
| `mem0_search` | 语义搜索 + 可选重排 (top_k ≤ 50) |
| `mem0_conclude` | 逐字存储事实（`infer=False`，不经过 LLM 提取） |

**熔断器**: 连续 5 次失败后暂停 API 调用 120 秒

**过滤范围**:
- 读: user_id 范围（跨会话召回）
- 写: user_id + agent_id 范围（归因）

### 其他提供商

| 提供商 | 大小 | 特点 |
|--------|------|------|
| Hindsight | 64KB | 知识图谱 + 实体解析 + 多策略检索，支持云端/本地 |
| Honcho | 424KB | 最复杂，CLI + client + session 模块 |
| OpenViking | 25KB | — |
| RetainDB | 32KB | — |
| Supermemory | 33KB | — |

### 内置记忆工具 (`tools/memory_tool.py`)

- 有界、文件支撑的记忆: MEMORY.md (agent notes) + USER.md (user profile)
- 条目分隔符: § (section sign)
- Session 开始时注入冻结快照；中途写入更新磁盘但不更新系统提示词（保留 prefix cache）
- 含注入和泄露扫描

## 数据流

```
Session 开始
  ↓
memory_provider.initialize()
  ↓
每次 API 调用前:
  prefetch(query) → 注入 <memory-context>
  ↓
工具调用:
  handle_tool_call(brv_query/mem0_search/fact_store/...)
  ↓
轮次结束:
  sync_turn(user, assistant)
  ↓
Session 结束:
  shutdown()
```

## 配置项

```yaml
# config.yaml
memory:
  provider: holographic  # byterover / mem0 / holographic / hindsight / ...
  mode: null
```
