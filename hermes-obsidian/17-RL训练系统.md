# RL 训练系统

> 基于 Atropos/Tinker 的强化学习自进化

## 概述

RL 训练系统使 Hermes 能通过强化学习自我改进。基于 Atropos 轨迹 API 协调环境交互，使用 Tinker 进行 LoRA 微调。

## 核心架构

```
HermesAgentBaseEnv (environments/hermes_base_env.py)
  ├── Agent Loop (agent_loop.py)
  ├── Atropos Trajectory API
  └── Tinker LoRA Training

TinkerAtroposTrainer (tinker-atropos/)
  ├── Rollout 采集
  ├── 优势计算
  ├── 梯度更新
  └── wandb 日志
```

## 实现原理

### HermesAgentBaseEnv (`environments/hermes_base_env.py`)

抽象基类，扩展 `atroposlib.BaseEnv`:

**两阶段运行**:
| 阶段 | 模式 | 用途 |
|------|------|------|
| Phase 1 | OpenAI server | VLLM, SGLang, OpenRouter, OpenAI API |
| Phase 2 | VLLM ManagedServer | 客户端工具调用解析器 |

子类实现: `setup()`, `get_next_item()`, `format_prompt()`, `compute_reward()`, `evaluate()`

配置 (`HermesAgentEnvConfig`): 工具集（启用/禁用/分布采样）、Agent 循环（最大轮次/温度/系统提示词）、终端后端、HuggingFace 数据集加载

### Agent Loop (`environments/agent_loop.py`)

`HermesAgentLoop` 使用标准 OpenAI 规范工具调用:
- 与任何返回 `ChatCompletion` + `tool_calls` 的服务器协作
- 工具执行通过 `handle_function_call()` (model_tools.py)
- 线程池 (默认 128 workers) 处理同步工具调用
- 返回 `AgentResult`: 完整对话历史、托管状态、使用轮次、每轮推理、工具错误

### Tinker-Atropos Trainer (`tinker-atropos/`)

`TinkerAtroposTrainer` 训练循环:

1. 注册 Atropos API 服务器
2. 获取 rollout 批次
3. 转换为 Tinker `Datum` 对象（带 padded logprobs 和 advantages）
4. 群组相对优势: `score - mean(scores)`
5. 通过 Tinker LoRA 客户端执行梯度步骤
6. 保存权重 + 创建新采样客户端
7. 日志 → wandb

**配置** (`TinkerAtroposConfig`):
- group_size: 16
- batch_size: 128
- lora_rank: 32
- learning_rate: 4e-5
- max_token_trainer_length: 2048
