# 浏览器工具

> 三后端 Web 自动化：本地 Chromium / Browserbase 云 / Camofox

## 概述

浏览器工具支持三种后端：本地 Chromium（零成本）、Browserbase（云端隐身）、Camofox（自托管指纹伪装）。通过可访问性树提供文本化页面表示。

## 核心架构

```
browser_tool.py (主入口)
  ├── Local Chromium (agent-browser CLI)
  ├── Browserbase (云端)
  └── Browser Use (云端)

browser_cdp_tool.py (CDP 直通)
browser_camofox.py (Camofox 集成)
```

## 实现原理

### 主浏览器工具 (`tools/browser_tool.py`)

| 后端 | 要求 | 特点 |
|------|------|------|
| Local Chromium | 无（默认） | headless, ariaSnapshot 文本表示 |
| Browserbase | `BROWSERBASE_API_KEY` + `BROWSERBASE_PROJECT_ID` | 代理、隐身、keepAlive |
| Browser Use | `BROWSER_USE_API_KEY` | 云端 |

**可访问性树**: 页面通过 `ariaSnapshot` 转为文本，元素通过 `@e1`, `@e2` 引用

### CloudBrowserProvider ABC (`tools/browser_providers/base.py`)

```python
class CloudBrowserProvider:
    provider_name() -> str
    is_configured() -> bool
    create_session(task_id) -> Dict  # → session_name, bb_session_id, cdp_url, features
    close_session(session_id)
    emergency_cleanup(session_id)
```

实现: BrowserbaseProvider, BrowserUseProvider, FirecrawlProvider

### CDP 直通 (`tools/browser_cdp_tool.py`)

`browser_cdp` 工具发送任意 Chrome DevTools Protocol 命令:
1. 连接 CDP URL (`BROWSER_CDP_URL` 或 `browser.cdp_url` 配置)
2. 可选附加目标: `Target.attachToTarget` 获取页面级 session
3. 分发 CDP 方法

这是主浏览器工具未覆盖操作的**逃生舱**: 原生对话框、iframe 求值、cookie/网络控制

### Camofox (`tools/browser_camofox.py`)

- 自托管 Node.js 服务器包装 Camoufox（Firefox 分支 + C++ 指纹伪装）
- 设置 `CAMOFOX_URL` 后所有浏览器操作路由到其 REST API
- 支持: VNC 查看、托管持久化（稳定浏览器配置文件）、session 管理
- `is_camofox_mode()`: `CAMOFOX_URL` 已设置且无 CDP 覆盖

## 配置项

```yaml
# config.yaml
browser:
  inactivity_timeout: 300
  command_timeout: 60
  record_sessions: false
  allow_private_urls: false
  cdp_url: null
  dialog_policy: auto
```

环境变量: `BROWSERBASE_API_KEY`, `BROWSERBASE_PROJECT_ID`, `BROWSER_CDP_URL`, `CAMOFOX_URL`
