# 记忆系统 — 架构

> 五层生命周期 + AMS 评分引擎 + 新陈代谢机制

## 概述

记忆系统是 Hermes 最核心的设计之一。黄金层只存指针，实际记忆在本地文件和技能中。记忆有明确的生命周期：从工作记忆逐步升级到核心记忆，或降级归档。AMS（自主记忆评分引擎）负责评分和路由决策。

## 核心架构

### 五层生命周期

```
working (工作记忆)
    ↓ session 结束时摘要
short_term (短期记忆, 带 TTL)
    ↓ 满足升级条件
long_term (长期记忆, 每 90 天复审)
    ↓ 满足升级条件 + 人工确认
core (核心记忆, 每 180 天复审)
    ↓ 任何层级都可以
archive (淘汰记忆, 保留 365+ 天)
```

### 黄金层（只存指针）

| 文件 | 内容 |
|------|------|
| `SOUL.md` | 核心身份 + 5 条原则 + 9 子系统路由规则 |
| `memory.md` | 紧凑记忆地图（4 段：核心层/生命周期层/检索层/治理层） |

**规则**: 存指针不存正文，优先升级/降级/归档而非无限增长

### 各层详情

#### 工作记忆 (`memory/working/`)

- 当前会话的临时任务状态
- **永不持久化原始内容**
- Session 结束时有价值洞察摘要到短期

**反模式**: 复制整个对话、"以防万一"保留、直接升级原始工具输出

#### 短期记忆 (`memory/short_term/`)

- 带强制 TTL 的近期上下文
- Frontmatter 字段: `id` (stm_YYYYMMDD_seq), `type`, `importance`, `stability`, `confidence`, `ttl`

**TTL 指南**:
| 类型 | TTL |
|------|-----|
| 假设 | P30D |
| 任务上下文 | P7D |
| 市场观察 | P7D |
| 跟进 | P14D |
| 中期分析 | P21D |

**自动归档**: AMS 在 TTL 到期时评估 — 近期被引用则延长 50%；已验证则考虑升级；否则归档

#### 长期记忆 (`memory/long_term/`)

- 已验证的可复用知识
- Frontmatter: `id` (ltm_YYYYMMDD_seq), `type` (research_conclusion/methodology/...), 证据链
- 每 90 天复审

**复审规则**:
- 综合分 < 0.50 → 降级到短期
- 综合分 ≥ 0.80 + 稳定性 ≥ 0.85 → 升级为核心候选

#### 核心记忆 (`memory/core/`)

- 最受保护层
- 变更要求: 综合分 ≥ 0.80 **且** 稳定性 ≥ 0.85
- Frontmatter: `importance: 0.9-1.0`, `stability: 0.85-1.0`, `confidence: 0.85-1.0`
- 每 180 天复审
- 综合分 < 0.75 → 降级到长期

#### 淘汰记忆 (`memory/archive/`)

6 种归档原因码:
- `TTL_EXPIRED` — TTL 到期
- `SUPERSEDED` — 被新信息取代
- `INVALIDATED` — 被现实证伪
- `MERGED` — 合并到其他记忆
- `DEPRECATED` — 不再相关
- `TASK_COMPLETE` — 任务完成

保留期: 最少 365 天；安全事件永久保留

## 实现原理

### AMS 评分引擎 (`memory/ams-engine.md`)

6 维评分（每维 0.0-1.0）:

| 维度 | 权重 | 说明 |
|------|------|------|
| Importance | 0.25 | 对核心目标的重要性 |
| Stability | 0.20 | 随时间保持真实的概率 |
| Reuse Value | 0.15 | 被需要的频率 |
| Time Sensitivity | 0.10 | 衰减速度（1.0=永恒） |
| Confidence | 0.15 | 正确性的确定程度 |
| Core Relevance | 0.15 | 与用户主要目标的直接关联 |

**综合分公式**:
```
composite = importance*0.25 + stability*0.20 + reuse_value*0.15
          + time_sensitivity*0.10 + confidence*0.15 + core_relevance*0.15
```

**路由决策矩阵**:
| 综合分 | 稳定性 | 路由 |
|--------|--------|------|
| ≥ 0.80 | ≥ 0.80 | Core |
| ≥ 0.80 | < 0.80 | Long-term |
| 0.50-0.79 | ≥ 0.60 | Long-term |
| 0.50-0.79 | < 0.60 | Short-term |
| 0.30-0.49 | — | Short-term (带 TTL) |
| < 0.30 | — | 丢弃 |

**特殊路由**:
- 关系密集 → Neo4j 候选
- 语义相似 → Qdrant 候选
- 可执行方法论 → Skills 候选
- 反馈/结果对 → 反馈闭环候选

### 新陈代谢机制

**升级触发（短期→长期）**: 全部满足: 综合分≥0.60, 稳定性≥0.60, 创建≥14天, 引用≥2次

**升级触发（长期→核心）**: 全部满足: 综合分≥0.80, 稳定性≥0.85, 长期层≥90天, 引用≥5个不同会话, 人工确认

**降级触发（长期→短期）**: 任一满足: 综合分<0.50, 稳定性<0.40, 90天未引用, 被取代

**定期复审**:
| 层级 | 周期 |
|------|------|
| Core | 180 天 |
| Long-term | 90 天 |
| Short-term | 30 天 / TTL |
| Working | Session 结束 |
| Archive | 365 天 |

### 记忆检索顺序（严格）

1. **指针层**: SOUL.md + memory.md
2. **关键词/标签**: fact_store + skills
3. **语义**: byterover + Qdrant
4. **图/时间线**: Neo4j

### 记忆对象 Schema

必填字段: `id`, `type`, `layer`, `importance`, `stability`, `tags`, `source`, `timestamp`, `ttl`, `status`, `pointer`, `content_summary`

生命周期状态: `working`, `short_term`, `long_term`, `core`, `archived`, `deprecated`

### 反馈闭环 (`memory/feedback-loop.md`)

7 步通用循环: 形成预期 → 定义检查点 → 采取行动 → 观察结果 → 比较 → 更新 → 记录

5 种结果类型:
| 结果 | 置信度变化 |
|------|-----------|
| CONFIRMED | +0.05 ~ +0.15 |
| PARTIALLY_CONFIRMED | -0.05 ~ +0.05 |
| INCONCLUSIVE | 0, 延长检查点 |
| CONTRADICTED | -0.15 ~ -0.30 |
| INVALIDATED | -0.30+, 归档 |

AMS 集成: 3+ 确认 + 稳定性≥0.60 → 升级长期；5+ 确认 + 稳定性≥0.85 → 核心候选；2+ 矛盾 → 归档

### 降噪音 (`memory/noise-monitor.md`)

5 类噪声: 冗余 / 低价值 / 过期 / 可指针化 / 超出范围

**100K 上下文 token 预算分配**:
| 类别 | 占比 |
|------|------|
| 系统提示词 | 3% |
| 角色/规则 | 2% |
| 核心记忆 | 0.5% |
| 任务相关召回 | 5% |
| 当前任务 | 15% |
| 对话历史 | 20% |
| 工具结果 | 10% |
| **推理预留** | **44.5%** |

4 级修复:
| 噪声率 | 状态 | 动作 |
|--------|------|------|
| 0-20% | 健康 | 无 |
| 21-35% | 升高 | 去重/指针化/摘要 |
| 36-50% | 高 | 截断历史、驱逐低重要性记忆 |
| >50% | 严重 | 强制压缩，只保留宪法+角色+当前任务+最近5轮 |
